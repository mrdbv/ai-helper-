# Booking Bot — Telegram-бот для запису клієнтів

MVP-бот для салонів краси, репетиторів, майстрів — клієнти записуються
самостійно через Telegram, власник отримує сповіщення й бачить розклад.

## Швидкий старт

1. Створіть бота через [@BotFather](https://t.me/BotFather), отримайте токен
2. Дізнайтесь свій Telegram ID через [@userinfobot](https://t.me/userinfobot)
3. Скопіюйте `.env.example` в `.env` і заповніть:
   ```
   cp .env.example .env
   ```
4. Встановіть залежності й запустіть:
   ```
   npm install
   npm start
   ```

## Структура

```
src/
  index.js          — точка входу, реєстрація команд
  db.js             — шар даних (JSON-файл db.json, легко замінити на Postgres)
  slots.js          — логіка генерації вільних часових слотів
  admin.js          — команди для власника: /today, /week, /cancel
  reminders.js      — щоденні нагадування клієнтам о 18:00
  scenes/
    bookingScene.js — покроковий діалог запису (wizard-сцена)
```

## Команди для клієнта
- `/start` — привітання
- `/book` — почати запис (послуга → дата → час → ім'я → телефон → підтвердження)
- `/my` — переглянути свої записи

## Команди для власника (тільки ADMIN_CHAT_ID)
- `/today` — записи на сьогодні
- `/week` — всі майбутні записи
- `/cancel <id>` — скасувати запис (клієнту прийде сповіщення)

## Що змінити під свій бізнес

1. Список послуг — у `src/db.js`, масив `defaultData.services`
2. Робочі години й вихідні — у `.env` (`WORK_START`, `WORK_END`, `DAYS_OFF`)
3. Назва бізнесу — `.env`, `BUSINESS_NAME`

## Наступні кроки після MVP

- Перенести дані з JSON-файлу на Postgres/Supabase, коли записів стане багато
- Додати редагування списку послуг прямо з Telegram (адмін-меню замість
  правки коду)
- Підтримка кількох майстрів в одному боті (розклад окремо на кожного)
- Веб-панель для власника замість команд у чаті
